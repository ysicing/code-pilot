# Claude Code 智能开发工作流系统

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

> 精简高效的Claude Code工作流系统，13个核心命令 + 5个专业角色，覆盖90%开发场景

## 🎯 核心理念

**KISS原则 + 专业分工 + 自动化质量**

本系统提供了**精简而强大的Claude Code工作流**，实现：

- **🎛️ 自动化质量门控**：实用质量标准，无阻塞性错误即可
- **🧠 专业化分工**：5个核心专家角色，13个实用命令
- **⚡ 轻量级流程**：去除冗余，专注核心开发任务
- **📊 智能路由**：自动判断问题类型，触发合适工具
- **🎯 实用主义**：避免过度工程化，解决实际问题

## 🚀 工作流对比

**传统方式**：手动协调多个工具
```bash
claude: 帮我实现用户认证 → 需要架构设计吗？ → 需要review吗？ → 需要测试吗？
# 多轮确认，认知负担重
```

**现在方式**：一键自动化工作流
```bash
/requirements-pilot "实现JWT用户认证系统"  # 快速开发
/kiro "实现JWT用户认证系统"                # 规范驱动  
/bmad-pilot "实现JWT用户认证系统"          # 企业级
# 自动化执行，质量保证，智能决策
```

## 📋 三大核心工作流

### 1. ⚡ Requirements-Pilot 快速开发工作流

**适用场景**：大多数功能开发、快速原型、小型项目  
**核心优势**：端到端自动化、快速迭代、质量门控

**流程**：
```
需求分析 → 代码实现 → 质量检查 → 测试验证 → 完成交付
```

### 2. 📋 Kiro 规范驱动开发

**适用场景**：复杂功能规范、系统架构设计  
**核心优势**：3阶段结构化、需求明确、设计完整

**流程**：
```
需求文档 → 设计文档 → 任务分解 → 实施计划
```

### 3. 🏢 BMAD-Pilot 企业级协作

**适用场景**：企业级系统、最高质量要求  
**核心优势**：AI团队协作、完整质量体系

**流程**：
```
产品规划 → 架构设计 → 开发实现 → 质量保证 → 发布管理
```

## 🛠️ 13个核心命令

### 📊 分析工具（4个）
- `/ask` - 系统架构师咨询
- `/story-breakdown` - 复杂功能分解
- `/plan` - 实施策略制定
- `/ultrathink` - MECE结构化深度分析

### ⚙️ 开发工具（5个）
- `/code` - 直接功能实现
- `/refactor` - 安全渐进式重构
- `/bugfix` - 系统化缺陷解决
- `/debug` - 深度调试分析
- `/optimize` - 性能优化专家

### 🔍 质量保证（2个）
- `/review` - 多维度代码审查
- `/test` - 智能测试策略

### 🎯 工作流选择指南
- **快速功能开发**：优先使用 `/requirements-pilot`
- **复杂功能规范**：使用 `/kiro` 进行3阶段规范创建
- **企业级项目**：使用 `/bmad-pilot` 完整团队协作
- **重大技术决策**：使用 `/ultrathink` 进行MECE深度分析

## 👥 5个专业角色

### 🏗️ 核心专家团队
- **architect** - 系统架构师，专注可扩展设计和技术决策
- **security** - 安全专家，专注威胁建模和防护策略
- **performance** - 性能专家，专注系统优化和瓶颈分析
- **frontend** - 前端专家，UI/UX设计与实现、现代组件开发
- **reviewer** - 代码审查专家，专注代码质量和最佳实践

### 🎯 智能路由机制
系统自动根据问题类型触发合适的专家：
- 架构设计问题 → `architect`
- 技术选择对比 → `/ultrathink`
- 前端相关需求 → `frontend`
- 代码质量问题 → `reviewer`

## 🚀 快速开始

### 1. 基础使用
```bash
# 快速实现功能
/requirements-pilot "实现用户注册登录功能"

# 获得架构建议  
/ask "如何设计高并发的消息队列系统？"

# 代码审查
/review "请审查这段API代码的安全性和性能"
```

### 2. 智能后缀
```bash
# 强制深度分析
"技术选型问题 -u"

# 快速模式（禁用自动触发）
"简单问题 -q"

# 详细调试模式
"bug分析 -d"
```

### 3. 工作流示例
```bash
# 场景：开发一个博客系统
/kiro "开发一个支持Markdown的博客系统"
# → 生成需求文档、设计文档、任务分解

# 场景：性能优化
/optimize "这个API响应很慢，请帮我优化"
# → 自动性能分析、瓶颈识别、优化建议
```
